# تحديث رحلة التمويل — بوابة الشركات

## ما تم تنفيذه

1. **واجهة رئيسية توضح رحلة التمويل كاملة**
   - رفع القوائم.
   - التقييم المالي.
   - استكمال المستندات.
   - مراجعة البنك.
   - الموافقة المبدئية/المشروطة.
   - الضمان عند الحاجة.
   - الموافقة النهائية والصرف.
   - الأقساط والمتابعة الشهرية.
   - قسم أسئلة شائعة يشرح الضمان والموافقة المشروطة والمتابعة.

2. **نموذج طلب تمويل منظم**
   - رقم السجل التجاري والنشاط وتاريخ التأسيس وعمر الشركة.
   - اسم مالك الشركة أو المفوض ورقم الهوية.
   - بيانات التواصل ومبلغ التمويل ومدته وغرضه.
   - المستندات الإلزامية:
     - القوائم المالية من خطوة التحليل.
     - كشف الحساب البنكي لآخر 6 أشهر.
     - السجل التجاري.
     - هوية مالك الشركة أو المفوض بالتوقيع.
   - المستندات الاختيارية:
     - خطة استخدام مبلغ التمويل.
     - دراسة الجدوى.
   - مستندات الضمان لا تُطلب في البداية، بل تظهر بعد الموافقة المبدئية عند الحاجة.

3. **صفحة متابعة تمويل احترافية**
   - شريط رحلة من التقديم إلى المتابعة الشهرية.
   - عرض الموافقة المشروطة والشروط وحالة كل شرط.
   - زر رفع الضمان بعد الموافقة المبدئية.
   - رسالة واضحة عند الموافقة النهائية.
   - سجل الصرف وجدول الأقساط والسداد.
   - طلبات البنك والتحديثات الدورية والإنذار المبكر.
   - رفع القوائم الشهرية أو كشف الحساب مباشرة للبنك.

4. **الضمانات**
   - الضمان خط حماية ثانٍ وليس بديلًا عن التدفق النقدي.
   - حساب القيمة المؤهلة بعد Haircut وتكاليف التسييل والخصم وحدود التركّز.
   - إظهار التغطية المطلوبة والحالية والعجز.
   - رفع مستندات الملكية والتقييم والتأمين والرهن حسب نوع الضمان.
   - الضمانات تظهر فقط بعد الموافقة المبدئية أو المشروطة.
   - نسب السياسة قابلة للتعديل من `config/collateral-policy.json`.

5. **الحساب المشترك بين البوابتين**
   - بيانات الشركة والسجل والنشاط تُحفظ في ملف الشركة.
   - الطلب والمستندات والضمانات والصرف والمتابعة مخزنة في قاعدة البيانات المشتركة.

## التحقق

- `npm run typecheck` نجح.
- `npm test` نجح: **22/22 اختبارًا**.
- `npm run build` نجح وأنتج نسخة Production.

## التشغيل

```bash
npm install
npm run dev
```

بوابة الشركات تعمل افتراضيًا على:

```text
http://localhost:3000
```

يجب وضع نفس `DATABASE_URL` المستخدم في بوابة البنك داخل `.env.local`، مع `SESSION_SECRET` مختلف خاص ببوابة الشركات.
